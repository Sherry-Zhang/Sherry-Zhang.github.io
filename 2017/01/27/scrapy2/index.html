scrapy学习(二)：scrapy基础使用

发表于 2017-01-27

初始化一个scrapy项目

使用命令行初始化，执行：

$ scrapy startproject [项目名]

初始的目录结构如下

[项目名]/
├── scrapy.cfg                # scrapy的部署配置文件
└── [项目名]                   # 项目的python块，我们的代码就放到这里
        ├── __init__.py       
        ├── items.py          # 项目的目标文件（要爬取的对象） 
        ├── middlewares.py     
        ├── pipelines.py      # 项目的管道文件（确定存储内容和方式）
        ├── settings.py       # 项目的配置文件
        └── spiders           # 项目的爬虫文件
                └── __init__.py

spiders

爬虫模块，负责的功能有：

  • 配置需要爬取的数据的来源(urls)
  • 配置爬取规则
  • 解析结构化数据

在spiders目录下面新建一个爬虫模块maoyanSpider.py。它必须继承 scrapy.Spider类， 且定义以下三个属性:

  • name: 爬虫的名字，必须唯一。通过控制台启动爬虫时，是根据这个名字来启动的。
  • start_urls: 初始爬取的url列表。
  • parse() response结果处理函数。负责解析返回的数据(response data)，提取数据(生成item)以及生成需要进一步处理的URL的 Request 对象。

settings.py

注释着很多常用的配置项，通过在其他文件中引入本文件的方式来使用这些配置项。默认打开的配置项：

robots.txt是遵循Robot协议的一个文件，它保存在网站的服务器中，它的作用是，告诉搜索引擎爬虫，本网站哪些目录下的网页不希望你进行爬取收录。在Scrapy启动后，会在第一时间访问网站的robots.txt文件，然后决定该网站的爬取范围。

由于我们并不是写搜索引擎，很多时候需要爬取的东西不是网站希望我们爬取的，是违反Robot协议的。因此settings.py里的这项值要改为False

运行爬虫程序

到项目根目录下，通过命令行运行爬虫 ：

$ scrapy crawl maoyan

根目录下生成了一个maoyanIndex.html的文件，正是我们下载的猫眼首页内容。
